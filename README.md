# journal-full-ayx

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面。这些页面以静态文件形式存放，不依赖特定后端服务，可被直接托管或访问。

## 目录结构

```
.
├── README.md          # 本文件
├── index.html         # 入口页面（可选）
├── pages/             # 存放各独立 HTML 页面
│   ├── example1.html
│   ├── example2.html
│   └── ...
└── assets/            # 公共资源（CSS、JS、图片等）
    ├── style.css
    └── script.js
```

- `pages/`：每个独立 HTML 页面均放置在此目录下，文件名可反映页面内容。
- `assets/`：包含多个页面可能共享的样式、脚本及媒体文件。

## 页面归档说明

- 各 HTML 页面均为独立完整文档，可直接在浏览器中打开。
- 页面之间无强关联，可根据需要单独访问或链接。
- 所有资源引用均采用相对路径，便于本地预览或部署到任意静态托管平台。

## 维护说明

- 新增页面：在 `pages/` 目录下创建新的 `.html` 文件，并确保引用资源路径正确。
- 更新页面：直接修改对应 HTML 文件，注意保持文件结构一致。
- 删除页面：移除 `pages/` 下的相关文件，并清理 `assets/` 中不再使用的资源。
- 建议在提交前使用本地浏览器预览，确认页面渲染正常。

## 部署

本仓库可直接通过 GitHub Pages 或其他静态托管服务部署。部署时，将仓库根目录作为站点根目录即可。
